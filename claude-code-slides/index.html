The Multi-Step Workflow

"Look at how X is done"

"Create a plan - don't code yet"
3. Implement
"Now implement the plan"
4. Verify
"Run tests and fix failures"

"Commit and create a PR"
The professional workflow.

- Don't let Claude jump straight to coding
- Research: understand existing patterns
- Plan: catch issues early
- Implement: focused execution
- Verify: quality check
- Commit: clean handoff

Good vs Bad Prompts

‚ùå Vague "Fix the login bug"
‚úì Better "Fix the null pointer in UserService.getById - add null handling"
Specificity wins.

- Vague prompt: Claude has to guess
- Better prompt: Claude knows exactly what to do
- Include: what, where, and constraints
- "Two minutes of clarity saves twenty minutes of back-and-forth"

Prompt Template

What What you want done
Where Which files/areas
Constraints What to preserve/avoid
Verify How to know it's done right
The formula for great prompts.

- What: the actual task
- Where: scope it to specific files/folders
- Constraints: don't break X, match Y style
- Verify: tests pass, build succeeds, etc.

Codebase-Aware Changes

"Rename User to Account across the codebase"
"Add TypeScript types to all /utils functions"
"Replace console.log with our logger utility"
Claude understands your whole codebase.

- Rename across files: Claude updates all imports
- Bulk updates: Apply changes to many files at once
- Pattern changes: Replace patterns consistently
- Be explicit about scope: "only in /src" or "entire repo"
